# v1.11.0 패치노트

**릴리즈일**: 2026-03-20

## 새 기능

### Bot/API 완전 분리 (아키텍처)
- Discord Bot과 NestJS API가 독립 프로세스로 분리되었습니다
- Bot은 Discord 이벤트 수신 + API HTTP 호출, API는 비즈니스 로직 + DB 전담
- API에서 Discord Gateway 완전 제거, Discord REST API로 전환
- 14개 슬래시 커맨드, 6개 이벤트 핸들러가 Bot으로 이동 완료
- Bot → API 통신을 위한 `@onyu/bot-api-client` SDK 라이브러리 추가

### AutoChannel 인터랙션 Bot 이동
- 자동방 버튼 인터랙션(auto_btn/auto_sub)이 Bot에서 처리됩니다
- Bot이 Discord 인터랙션을 수신하고, API에 비즈니스 로직을 위임하는 구조로 전환
- `POST /bot-api/auto-channel/button-click`, `POST /bot-api/auto-channel/sub-option` 엔드포인트 추가

### 음악 재생 Lavalink 전환
- discord-player에서 Lavalink v4 + Kazagumo v3로 음악 재생 라이브러리 전환
- Docker Compose에 Lavalink 서비스 추가
- Spotify/YouTube/SoundCloud 소스 지원

### 음성 활동 추가 데이터 수집
- 스트리밍, 카메라, 스피커 음소거 상태를 추가 수집합니다
- 게임 활동 정보(게임명, applicationId) 수집 인프라 추가
- Co-Presence 스냅샷에 멤버별 게임 활동 정보 포함

### 다국어(i18n) 지원
- 웹 대시보드 전체 한국어/영어 로컬라이징 완료 (next-intl)
- 봇 메시지(슬래시 커맨드 응답, Embed) 한국어/영어 지원
- 시간/등급 포맷 함수 i18n 전환 (14개 컴포넌트)
- 브라우저 Accept-Language 자동 감지 + 쿠키 기반 언어 전환

### Timezone(UTC) 지원
- 데이터 저장 구조가 UTC 기반으로 전환되었습니다
- 프론트엔드에서 사용자 로컬 타임존으로 자동 변환

### 상용화 인프라
- 온보딩 페이지, 이용약관, 개인정보처리방침 페이지 추가
- 데이터 보존 정책 설정 기능 추가
- 웹 대시보드 모바일 반응형 개선

## 변경

### 아키텍처 리팩토링
- Pragmatic DDD 가이드 준수 모듈 재구조화 (Domain Entity ↔ ORM Entity 분리)
- 레거시 중복 파일 제거 및 공통 유틸/타입 통합
- 코드 스타일 가이드 ESLint 규칙 동기화
- CoPresence/Monitoring 스케줄러가 Bot Push 방식으로 전환

### 성능 최적화
- Co-Presence 배치 저장 + 단방향 전환으로 DB 부하 감소
- 비활동 회원 자동조치 동시성 제한 병렬 처리

### 자동방 설정 페이지 안정화
- Discord 안내 메시지 전송 실패 시 에러 핸들링 추가 (DB 저장은 유지)
- 유효하지 않은 이모지 입력 검증 추가 (Invalid Form Body 방지)
- 프론트엔드 검증 강화 (안내 메시지, 버튼 필수, 하위 선택지 검증)
- 400 에러 응답 시 API validation 메시지 표시 개선

### StatusPrefix 핸들러 리팩토링
- 단일 메서드에서 handleApply/handleReset/setNickname/replyError로 분리

### CI/CD 개선
- Bot, bot-api-client에 eslint.config.mjs 추가
- Bot vitest.config.ts 추가
- API/Bot/bot-api-client import 정렬 일괄 수정
