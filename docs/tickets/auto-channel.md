# 자동방 생성 (Auto Channel)

## 기능명
자동방 생성 (Auto Channel Creation with Button Interaction)

## 도메인
- `voice` (트리거 감지, 대기방/확정방 관리, 세션 추적 통합)
- `channel` (Discord 채널 CRUD)
- `web` (설정 UI)

## 요구사항 요약

디스코드 서버에서 트리거 채널 입장 → 대기방 생성 → 버튼 클릭으로 확정방 전환하는 2단계 자동방 생성 기능.
웹 대시보드에서 트리거 채널, 버튼 구성, 네이밍 규칙 등을 설정할 수 있다.

## 전체 흐름

```
[웹 설정 저장] → 트리거 채널에 안내 메시지(설명 + 버튼) 전송
       ↓
[사용자 트리거 채널 입장] → 대기방 생성 → 사용자 이동
       ↓
[안내 메시지에서 버튼 클릭]
  ├─ 하위 선택지 없음 → 대기방 이름/카테고리 변경 → 확정방으로 전환
  └─ 하위 선택지 있음 → Ephemeral 버튼 표시 → 선택 후 확정방으로 전환
       ↓
[확정방부터 세션 추적 시작] (대기방은 집계 제외)
       ↓
[모든 사용자 퇴장 시] → 채널 즉시 삭제
```

## 상세 요구사항

### 1. 트리거 채널
- 서버당 여러 트리거 채널 설정 가능
- 사용자가 트리거 채널에 입장하면 대기방 생성 후 이동
- 이미 생성한 자동방이 있어도 항상 새 방 생성

### 2. 대기방
- 별도 네이밍 템플릿 (웹에서 설정, 예: `⌛ {username}의 대기방`)
- 세션 추적 대상에서 제외 (집계하지 않음)
- 모든 사용자 퇴장 시 즉시 삭제

### 3. 안내 메시지 & 버튼
- 트리거 채널에 안내 메시지를 전송 (설명 텍스트 + 버튼 목록)
- 안내 메시지는 웹에서 설정 저장 시 전송/갱신
- Discord Button Component로 구현
- 버튼별 설정 항목:
  - 버튼 라벨 (이름)
  - 이모지
  - 생성 카테고리 (확정 시 이동할 카테고리)

### 4. 하위 선택지
- 특정 버튼에만 하위 선택지를 추가할 수 있음
- 버튼 클릭 시 Ephemeral 메시지로 추가 버튼 표시
- 선택지에 따라 채널명만 변경 (카테고리는 동일)
- 예: 스팀 → [일반] [경쟁] 선택

### 5. 확정방 전환
- 대기방의 이름과 카테고리를 변경하여 확정방으로 전환 (삭제+재생성 아님)
- 확정방 네이밍: `{username}` + 자동 순번 (동일 이름 존재 시만 순번 부여)
  - 예: "Onyu의 오버워치", "Onyu의 오버워치 2"
- 확정방 전환 시점부터 기존 voice 도메인 세션 추적 시스템과 통합
- 모든 사용자 퇴장 시 즉시 삭제

### 6. 채널 권한
- 생성자에게 특별 권한 부여하지 않음 (서버 기본 권한 적용)

### 7. 채널 수 제한
- 제한 없음

### 8. 웹 설정 UI
- 위치: 대시보드 > 서버 설정 > 자동방 설정 섹션
- 설정 가능 항목:
  - 트리거 채널 선택
  - 대기방 네이밍 템플릿
  - 안내 메시지 텍스트
  - 버튼 목록 (라벨, 이모지, 카테고리)
  - 버튼별 하위 선택지 (라벨, 이모지, 채널명 접미사)
