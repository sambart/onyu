# Voice 유저플로우

> 일반 길드 멤버가 `/my/voice` 마이페이지에서 본인 음성 활동 통계를 조회하는 흐름. `apps/web/app/my/voice/`(페이지·컴포넌트), `apps/api/src/channel/voice/presentation/me-voice.controller.ts`(엔드포인트), `apps/api/src/channel/voice/application/me-profile.service.ts`(비즈니스 로직)가 연동된다.
>
> 🔒 마커: 본인 식별(PII), 타인 데이터 차단(권한), JWT 검증 줄.
>
> 관련 PRD: [voice.md](../prd/voice.md) F-VOICE-050, F-VOICE-051, F-VOICE-052

---

## UF-VOICE-MY-001: 마이페이지 진입

### 입력
- 사용자가 헤더 내비게이션 또는 로그인 후 일반 멤버 라우팅을 통해 `/my/voice` 경로에 접근

### 처리
1. (공통 전제: 세션 쿠키 존재 여부 확인 — 없거나 만료 시 `/auth/discord`로 리다이렉트)
2. 세션이 유효하면 `/my/voice` 페이지 진입 허용 — 🔒 운영자 사이드바(`DashboardSidebar`) 레이아웃 적용 없이 독립 레이아웃으로 렌더 (권한 가드 `GuildMembershipGuard` 미적용 — 운영 권한 불필요, 사전 승인)
3. 페이지 마운트 시 `GET /api/users/me/voice/guilds` 호출하여 본인이 음성 활동 기록을 보유한 길드 목록을 로드 — 🔒 API가 JWT `sub`에서 userId를 강제 추출하여 본인 데이터만 조회 (PII — 사전 승인)
4. 응답 수신까지 로딩 상태 렌더

### 출력
- 성공: 활동 길드 목록을 수신하여 다음 단계(UF-VOICE-MY-002 또는 UF-VOICE-MY-003)로 전이
- 실패: 오류 안내 상태 렌더 (네트워크·서버 오류)

### 엣지케이스
- 세션 쿠키 없음 또는 만료 → `/auth/discord`로 리다이렉트 (UF-AUTH-001 재진입)
- 토큰 만료 중 API 401 응답 → 로그인 재유도
- `GET /api/users/me/voice/guilds` 네트워크 오류 → 오류 상태 렌더, 재시도 가능
- `/my/voice` 직접 URL 접근 시 로그인 여부와 무관하게 세션 검사를 선행하여 비로그인 사용자는 리다이렉트

## UF-VOICE-MY-002: 활동 기록 없음 (빈 상태)

### 입력
- UF-VOICE-MY-001에서 `GET /api/users/me/voice/guilds` 응답이 빈 배열(`[]`)로 반환

### 처리
1. 활동 길드 목록이 비어 있음을 확인
2. 길드 선택 드롭다운과 기간 선택, 통계 컴포넌트를 렌더하지 않음
3. 빈 상태 안내 화면을 렌더

### 출력
- 아직 음성 활동 기록이 없음을 안내하는 화면 렌더 (Discord 서버에서 음성 채널에 참여하면 통계가 누적된다는 맥락 포함)

### 엣지케이스
- 사용자가 여러 서버에 속해 있으나 어느 서버에서도 음성 활동이 없으면 동일하게 빈 상태 렌더
- 봇이 서버에 참여 중이나 해당 사용자가 음성 채널에 접속한 이력이 없는 경우에도 빈 상태
- 데이터 보존 정책(90일)에 의해 오래된 기록이 삭제된 후 현재 활동 기록이 없는 경우에도 빈 상태 렌더

---

## UF-VOICE-MY-003: 활동 길드 선택 및 초기 통계 로드

### 입력
- UF-VOICE-MY-001에서 `GET /api/users/me/voice/guilds` 응답이 1개 이상의 길드를 포함

### 처리
1. 첫 번째 길드를 기본 선택 상태로 설정
2. 기본 기간을 15일로 설정
3. `GET /api/users/me/voice/profile?guildId={첫번째guildId}&days=15` 를 호출하여 본인 통계를 로드 — 🔒 API가 JWT `sub`에서 userId를 강제 추출하여 요청한 guildId 내 본인 데이터만 반환 (PII — 사전 승인)
4. 응답 수신까지 통계 영역에 로딩 상태 렌더
5. 200 OK 응답 수신 시 `MeProfileData`를 하위 컴포넌트에 전달하여 렌더:
   - 요약 카드: 서버 내 순위, 총 음성시간, 활동일수, 일평균 접속
   - 마이크 통계 카드: 마이크 ON/OFF 시간, 마이크 사용률, 혼자 있던 시간
   - 일별 추이 바 차트: 최근 N일간 일별 음성 시간
   - 피크 요일 카드: 가장 활발한 요일, 주 평균 접속 시간
   - 뱃지 섹션: 활동 뱃지 목록
   - 제외 채널 안내: 통계에서 제외된 채널 목록 (있을 때만 표시)

### 출력
- 길드 선택 드롭다운 + 기간 선택 버튼(7일/15일/30일) + 전체 통계 컴포넌트 렌더
- 선택된 길드 이름 및 아이콘이 드롭다운에 표시 (Discord REST 정보 획득 시)

### 엣지케이스
- Discord REST 실패로 guildName/guildIcon이 null인 경우 → guildId를 대체 표시
- 길드 목록이 1개뿐이면 드롭다운 대신 고정 라벨로 표시하거나, 드롭다운을 비활성화할 수 있음 (구현 단계 결정)
- 첫 번째 길드에 대한 프로필 API가 204를 반환하는 경우 → UF-VOICE-MY-005(해당 기간 활동 없음) 처리
- `GET /api/users/me/voice/profile` 네트워크 오류 → 오류 안내, 재시도 가능 (길드 목록은 유지)

## UF-VOICE-MY-004: 길드 변경

### 입력
- 사용자가 길드 선택 드롭다운에서 다른 길드를 선택

### 처리
1. 선택된 guildId를 로컬 상태에 반영
2. 현재 기간(days) 값을 유지한 채 `GET /api/users/me/voice/profile?guildId={선택한guildId}&days={현재days}` 재호출 — 🔒 API가 JWT `sub`에서 userId를 강제 추출 (PII — 사전 승인)
3. 이전 통계 컴포넌트 영역에 로딩 상태를 표시하면서 응답 대기
4. 응답 수신 시 통계 컴포넌트를 새 데이터로 갱신

### 출력
- 선택한 길드의 본인 통계로 전체 통계 영역 갱신

### 엣지케이스
- 길드 변경 중 사용자가 다시 다른 길드를 선택 → 이전 요청은 취소하거나 무시하고 최신 선택에 대한 응답만 반영 (경쟁 상태 방지)
- 선택 길드의 프로필 API가 204를 반환하는 경우 → UF-VOICE-MY-005 처리
- 네트워크 오류 → 오류 안내, 이전 통계를 유지하거나 오류 상태로 전환

## UF-VOICE-MY-005: 기간 변경

### 입력
- 사용자가 기간 선택 버튼(7일 / 15일 / 30일) 중 하나를 클릭

### 처리
1. 선택된 days 값을 로컬 상태에 반영
2. 현재 선택된 guildId를 유지한 채 `GET /api/users/me/voice/profile?guildId={현재guildId}&days={선택한days}` 재호출 — 🔒 API가 JWT `sub`에서 userId를 강제 추출 (PII — 사전 승인)
3. 통계 영역에 로딩 상태 표시하면서 응답 대기
4. 응답 수신 시 통계 컴포넌트를 새 기간 데이터로 갱신

### 출력
- 선택한 기간 기준 본인 통계로 전체 통계 영역 갱신
- 기간 선택 버튼에서 현재 선택 상태 시각적으로 표시

### 엣지케이스
- 7/15/30 이외의 days 값이 전달될 경우 API가 400 Bad Request 반환 → 오류 안내 (정상 UI 흐름에서는 발생하지 않으나 직접 API 호출 시 방어)
- 선택한 기간 내 활동 기록이 없는 경우(204 No Content) → 해당 기간 활동 없음 안내 표시
- 기간 변경 중 길드 변경이 연속으로 발생하는 경우 → 최신 요청 응답만 반영

## UF-VOICE-MY-006: 해당 기간 활동 없음

### 입력
- UF-VOICE-MY-003, UF-VOICE-MY-004, UF-VOICE-MY-005 중 `GET /api/users/me/voice/profile` 호출 결과 API가 204 No Content를 반환

### 처리
1. 204 응답을 감지
2. 통계 컴포넌트 영역을 비움
3. 해당 기간 동안 음성 활동이 없음을 안내하는 상태 렌더

### 출력
- 해당 기간 활동 없음 안내 화면 (길드 선택 드롭다운 + 기간 선택 버튼은 유지)

### 엣지케이스
- 다른 기간을 선택하면 UF-VOICE-MY-005로 전이하여 새 기간 데이터를 조회
- 다른 길드를 선택하면 UF-VOICE-MY-004로 전이
- 해당 길드에서 본인 데이터가 아예 없는 경우(다른 사람의 guildId를 URL로 주입해도 본인 데이터가 없으면 동일하게 204 반환 — 보안 의도된 동작)

## UF-VOICE-MY-007: 본인 음성 통계 조회 (데이터 상세)

> UF-VOICE-MY-003의 세부 처리 확장 — 개별 컴포넌트의 입력/출력 동작을 명세한다.

### 입력
- `GET /api/users/me/voice/profile` 200 OK 응답 수신 (`MeProfileData` 포함)

### 처리

**요약 카드 (`MeSummaryCards`)**:
1. `rank` / `totalUsers` 에서 서버 내 순위와 전체 유저 수를 렌더 — 🔒 순위는 해당 길드의 전체 유저 대비 본인 순위 단일 숫자이며, 개별 유저 정보를 포함하는 리더보드 화면은 렌더하지 않음 (권한 — 타인 데이터 비노출, 사전 승인)
2. `totalSec` → 총 음성 시간(시·분 단위로 포맷)
3. `activeDays` → 활동일 수
4. `avgDailySec` → 일평균 접속 시간

**마이크 통계 카드 (`MeMicStatsCard`)**:
1. `micOnSec` / `micOffSec` → 마이크 ON/OFF 시간
2. `micUsageRate` → 마이크 사용률(%)
3. `aloneSec` → 혼자 있던 시간

**일별 추이 차트 (`MeDailyChart`)**:
1. `dailyChart` 배열에서 날짜별 `durationSec`을 바 차트로 렌더
2. 활동이 없는 날짜는 빈 바(0)으로 표시 또는 회색 처리

**피크 요일 카드 (`MePeakDayCard`)**:
1. `peakDayOfWeek` → 가장 활발한 요일
2. `weeklyAvgSec` → 주 평균 접속 시간

**뱃지 섹션 (`MeBadgeSection`)**:
1. `badges` 배열 → 획득한 활동 뱃지 목록 렌더
2. 빈 배열이면 뱃지 섹션을 숨기거나 미획득 상태 안내

**제외 채널 안내 (`MeExcludedChannelBanner`)**:
1. `excludedChannels` 배열이 비어 있으면 컴포넌트 자체를 렌더하지 않음
2. 배열에 항목이 있으면 통계에서 제외된 채널 안내 배너 렌더

### 출력
- 각 컴포넌트가 `MeProfileData` 필드를 소비하여 화면을 렌더
- 타인 데이터·리더보드·co-presence·비활동 분류는 이 화면에 일절 렌더하지 않음

### 엣지케이스
- `badges` 빈 배열 → 뱃지 영역 비어 있음 표시 또는 섹션 숨김 (구현 단계 결정)
- `excludedChannels` 빈 배열 → `MeExcludedChannelBanner` 미렌더
- `dailyChart` 배열이 비어 있음 → 차트 빈 상태 렌더 (204 응답 시는 이미 UF-VOICE-MY-006에서 처리되므로 200 + 빈 배열은 엣지 케이스)
- `rank`가 null 또는 산출 불가 → 순위 표시를 생략하거나 계산 불가 상태 표시
- `peakDayOfWeek`가 null(활동이 하나도 없는데 200이 온 경우 비정상) → 피크 요일 미표시 처리

## UF-VOICE-MY-008: 일반 멤버 로그인 후 라우팅

> 운영 권한 없는 사용자가 로그인 후 서버 선택 화면에서 선택 가능한 길드가 없을 때의 안내 흐름.

### 입력
- 사용자가 Discord OAuth 로그인을 완료했으나, JWT에 포함된 운영 권한 보유 길드 목록(`guilds`)이 비어 있음
- 현재 select-guild 화면에서 선택 가능한 서버가 없는 상태로 진입

### 처리
1. 로그인 완료 후 서버 선택(`select-guild`) 화면 또는 목적지 화면에 도달
2. 운영 권한 길드가 없어 기존 대시보드 진입이 불가능한 상태를 감지
3. 마이페이지(`/my/voice`)로 이동할 수 있는 안내를 제공
4. 사용자가 안내에 따라 `/my/voice`로 이동하면 UF-VOICE-MY-001 흐름 진입

### 출력
- 마이페이지 진입 안내 (운영 권한은 없지만 음성 통계 조회는 가능하다는 맥락)
- 운영 권한이 있는 길드가 생기면 서버 선택에서 해당 서버를 선택할 수 있음

### 엣지케이스
- 운영 권한은 없지만 음성 활동 기록도 없는 사용자 → `/my/voice` 진입 후 UF-VOICE-MY-002(빈 상태) 렌더
- 운영 권한이 있는 길드도 있고 음성 활동도 있는 사용자 → 헤더 내비를 통해 마이페이지와 운영 대시보드를 각각 독립적으로 이용 가능
- 로그인 상태에서 목적지 쿠키가 `/my/voice`였던 경우 → 로그인 완료 후 `/my/voice`로 직접 전이 (UF-AUTH-001 목적지 쿠키 흐름 연계)
