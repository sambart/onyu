# Onyu 전용 디스코드 길드 — IA (정보구조) 청사진

> **목적**: onyu 제품 커뮤니티/지원 길드의 채널·카테고리·역할·권한 청사진.
> **상태**: 1차 확정 (설계 완료) · **규모**: 중규모(~수백명) 대상 · **봇**: onyu 풀가동
> **작성일**: 2026-06-19

이 문서는 길드를 "처음 찍어낼 때"의 기준 구조다. 추후 MCP(또는 수동)로 이 구조를 그대로 생성한다.

---

## 1. 역할(Role) — 위계 (위 = 높음)

| # | 역할 | 부여 방식 | 핵심 권한 / 비고 |
|---|------|-----------|------------------|
| 1 | **onyu (봇)** | 봇 자동 | Manage Channels/Roles/Messages 등 운영봇 권한. 관리자 **바로 아래**에 배치 (자기보다 위 역할은 못 건드림) |
| 2 | **관리자** | 수동 | Administrator |
| 3 | **운영진(모더)** | 수동 | 메시지/멤버 관리 + 운영진 카테고리 접근 |
| 4 | 🟨 **후원자/부스터** | 부스트·후원 | 정회원 + α (선택 — 초기 생략 가능) |
| 5 | **정회원** | 규칙 동의 게이트 통과 시 | 커뮤니티 전체 접근 |
| 6 | **비활동** | 봇 자동 (inactive-member) | 정회원과 분리·표식용. ※ newbie 기능과 무관 |

> **미인증 상태**는 별도 역할 없이 `@everyone`(게이트 전)으로 처리한다.
> `신규` 역할 및 onyu **newbie 연동 인증은 사용하지 않는다** (아래 §4 참조).

## 3. 권한 매트릭스 (카테고리 단위)

채널은 카테고리 권한을 상속한다. 예외만 채널별로 override.

| 채널군 | @everyone (미인증) | 정회원 | 운영진 | 관리자 |
|--------|--------------------|--------|--------|--------|
| 시작하기 (환영/규칙/공지) | 보기 ✅ (규칙 동의 버튼만 상호작용) | 보기 | 보기 | 전체 |
| └ `문의하기` (공개 문의) | 보기 + **쓰기** ✅ (미인증도 질문 가능) | 보기/쓰기 | 보기/쓰기 | 전체 |
| 커뮤니티 · 음성/게임 · onyu 리포트 · 지원 | ❌ 숨김 | 보기/쓰기 ✅ | 보기/쓰기 | 전체 |
| 운영진 (운영-일반/모니터링/비활동관리) | ❌ | ❌ | ✅ | ✅ |
| 감사-로그 | ❌ | ❌ | 보기만 | ✅ |

**동선**: 가입 → `시작하기`만 보임 → `규칙-rules`의 동의 버튼 클릭 → `정회원` 부여 → 커뮤니티 전체 오픈.

## 4. 인증 게이트 — onyu 봇 신규 기능 (후속 구현 항목)

onyu **newbie 기능은 인증에 사용하지 않는다.** 대신 onyu 봇에 **별도의 "규칙 동의 버튼-역할" 기능**을 신규 구현한다 (Carl-bot 등 외부 유틸리티 봇 미사용).

**요구사항 (초안 — 추후 PRD/`/feat-implement`로 구체화)**:
- `규칙-rules` 채널에 "동의" 버튼(또는 리액션) 1개 게시 (sticky)
- 사용자가 클릭 → `정회원` 역할 자동 부여
- 이미 `정회원`이면 무시 (멱등)
- (선택) 부여 로그를 `모니터링-logs` 또는 `감사-로그`에 기록

> 이 기능은 본 IA 확정과 **별개 작업**이다. 길드 구조는 이 기능 없이도 생성 가능하며, 기능 구현 전까지는 운영진이 수동으로 `정회원`을 부여할 수 있다.

## 5. 성장 시 분리 경로 (지금은 안 만듦)

- 멤버 증가 시: 후원자 역할 + 전용 채널 활성화
- 리포트 활발 시: `활동-리포트` ↔ `랭킹-순위` 추가 세분화
- 운영 복잡 시: `모니터링-logs` ↔ 기능별 로그 채널 분리
- 게임 활발 시: 고정 음성방 / 게임별 채널 추가
